## アプリ名
  Letteo(レッテオ)

## サービス概要
  Letteoは、演出付きの「デジタル手紙」を作って送れるWebアプリです。  
  テンプレートを選んでメッセージを入れるだけで、  
  受け取る側はアニメーションとともに手紙を開く体験を楽しめます。
  気軽さと、ちょっとした特別感を両立したコミュニケーションを提供します。

## このサービスへの思い・作りたい理由
  ここ数年で私自身が結婚と出産（夫として）を経験した際、知人への報告するときにLINEで気軽に伝えましたが、もう少しだけかしこまった形で伝えられたらと思う場面がありました。LINEの「ギフトメッセージ」機能のように、ギフトそのものは不要でいいので、メッセージだけを特別に演出できるアプリがあれば便利だと感じています。また、手紙や写真を封筒から取り出すときのワクワク感や温かみのある体験をweb上でも再現したいという思いが、このサービスを作ろうと思ったきっかけです。

## ユーザー層について
  若年層（18〜35代）：SNSやメッセージアプリでのコミュニケーションに慣れており、デザインや表現手段を楽しむ傾向があるため、誕生日や記念日、友人同士のちょっとしたやり取りで利用されやすいと想定しています。また、20代後半から30代前半は結婚や出産報告が多い層だと思うのでユーザー層に若年層を設定しています。
  
## サービスの利用イメージ
1. ユーザー登録 / ログイン
2. テンプレートを選択
3. メッセージを入力
4. 手紙を保存して公開リンクを発行
5. URL(password)を相手に送信
6. 受信者がアニメーションとともに手紙を閲覧

## ユーザーの獲得について
  QiitaやZennで技術記事を投稿。ブログ記事で短いガイド記事を作成。（検索されやすいワード「デジタル 手紙 アプリ」を設定）
  SNS広告（基本的にはXを使用）
  知人に紹介。
  RANTEQコミュニティでの紹介。

## サービスの差別化ポイント・推しポイント
- **手紙体験に特化したUI**
  - メッセージカードだけでなく、封筒・便箋を含めた「手紙体験」を重視
  - 開封や表示の流れを含めて、気持ちが伝わる体験を設計

- **デザインの自由度**
  - 文字配置・フォント・サイズ・色などを調整可能
  - 便箋やカードのレイアウトを活かした表現ができる

- **受け取り体験の演出**
  - 受信ページでアニメーションを用いた演出を提供
  - ただ読むだけでなく、体験として楽しめる構成

- **手軽さと特別感の両立**
  - URLを送るだけで閲覧可能
  - 気軽に使える一方で、記念日や節目にも使える


## 主な機能(MVP)

- ユーザー登録 / ログイン（Devise）
- 手紙テンプレートの選択（便箋 / メッセージカード）mvpでは3パターンのみ作成
- テキスト入力（フォント / サイズ / 色は標準設定）
- 手紙の作成・保存
- 公開リンクの生成
- パスワード付き閲覧
- 受信ページでのアニメーション表示（Lottie）

## 今後実装予定の機能(本リリース)
- フォント / サイズ / 色の選択機能
- 手紙へ画像添付機能
- テンプレート(便箋/メッセージカードの増設)
- 開封演出の増設(lettiefilesより)
- 送信機能(mail,line.QRコードなど)
- ユーザーのパスワード再設定機能実装
- お問い合わせ機能

## 使用する技術スタック

- Backend: Ruby on Rails 7.1.4 / Ruby 3.2.3
- Database: PostgreSQL
- Frontend: Tailwind CSS / Stimulus
- Animation: lottie-web
- Authentication: Devise
- File Upload: ActiveStorage(本リリース)
- Deploy: Render
